# JEE Prep Web App - Context Document

## Overview
This document serves as the technical specification for building a web-based JEE preparation application that works entirely offline with AI-processed content packages.

## AI Processing Capabilities

### Input Processing
- **PDF Parsing**: Complete extraction of text and images from educational PDFs
- **Multi-file Correlation**: Link questions, solutions, and hints across separate documents
- **Image Analysis**: Visual content description and spatial relationship mapping
- **Content Structuring**: Organization into standardized JSON formats

### Output Generation
- **Structured Question Data**: Questions with options, answers, and metadata
- **Solution Details**: Step-by-step explanations with formula references
- **Hint Systems**: Progressive guidance for difficult problems
- **Image Descriptions**: Textual representation of diagrams when images unavailable
- **Cross-referencing**: Links between related content across documents

## File Structure (.wbp - Workbook Package)

### Package Contents
```
workbook_name.wbp (ZIP archive)
├── manifest.json          # Package metadata and version info
├── questions.json         # Main question database
├── solutions.json         # Detailed solutions
├── hints.json            # Progressive hints
├── metadata.json         # Source and categorization data
├── assets/               # Optional: Actual image files
│   ├── diagrams/
│   └── figures/
└── cross_references.json # Mapping between Qs, solutions, hints
```

## Web App Architecture

### Core Components
1. **Package Manager**: Handles .wbp file import/export
2. **Question Display**: Renders questions with images/diagrams
3. **Practice Engine**: Manages timed sessions and progress tracking
4. **Analytics Dashboard**: Performance insights and statistics
5. **Local Storage**: IndexedDB for offline data persistence

### State Management
```javascript
// Global application state
const appState = {
  packages: [],           // Imported .wbp packages
  questions: [],          // All available questions
  currentSession: null,   // Active practice session
  userProgress: {},       // Question attempt history
  settings: {}            // User preferences
};
```

### Data Flow Logic
1. **Import**: User drops .wbp file → App extracts JSON data → Data stored in IndexedDB
2. **Browse**: Query IndexedDB for questions by topic/difficulty/tags
3. **Practice**: Select questions → Create session → Track performance
4. **Review**: Access solutions/hints → Update progress tracking

## UI/UX Requirements

### Main Views
1. **Dashboard**: Package overview and quick access
2. **Question Bank**: Filterable/searchable question library
3. **Practice Mode**: Timed question sessions
4. **Review Center**: Solution viewing and performance analysis
5. **Settings**: Configuration and export options

### Interaction Patterns
- **Progressive Disclosure**: Hints revealed step-by-step
- **Immediate Feedback**: Answer validation with explanation
- **Performance Tracking**: Real-time accuracy and timing metrics
- **Adaptive Difficulty**: Question selection based on user performance
